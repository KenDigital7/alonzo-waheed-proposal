# Improved Build Prompt — Living Life Balance Proposal Page

> Use this prompt to generate the single-page React proposal artifact. It supersedes the original draft prompt. Copy everything below the horizontal rule.

---

Build a visually striking, single-page business proposal for **Living Life Balance**, a personal brand and wellness coaching company founded by Alonzo Waheed. This is a professional services proposal — it should feel premium, warm, and grounded in Black wellness culture. Editorial meets community-rooted. Not corporate cold. Not SaaS.

---

## Aesthetic Direction

- **Color palette:** Deep charcoal (`#1C1C1C`), warm off-white/cream (`#F5F0E8`), terracotta/amber accent (`#C06D38`), and a muted warm taupe for secondary backgrounds (`#E8E0D4`). No purple gradients. No blue tones.
- **Typography:** Pair a bold serif display font with a clean humanist sans-serif body. Use Playfair Display (display/headings) + DM Sans (body/UI). Load from Google Fonts.
- **Layout:** Generous white space. Wide-margin section containers. No cramped cards or tight grid gutters. Sections feel like spreads, not panels.
- **Dividers:** Thin horizontal rules in terracotta or a faint warm neutral — subtle, not decorative noise.
- **Tone signal:** If it feels like a wellness magazine feature profile crossed with a founder pitch deck, the aesthetic is right. If it feels like a SaaS pricing page or a corporate consulting deck, adjust.

---

## Proposal Structure

Build each section as a clearly delineated, full-width scroll section with a semantic `id` for smooth-scroll navigation.

## Technical Requirements

- Single-page React component using **Tailwind CSS** utility classes only — no custom CSS files, no CSS-in-JS
- All content is **static** — no data fetching, no APIs, no external dependencies beyond Google Fonts
- **Fully responsive** — mobile-first layout; all multi-column layouts stack cleanly on small screens
- **Smooth scroll** navigation between sections (use `scroll-behavior: smooth` or JS-based smooth scroll)
- **Scroll-triggered animations** on section entry — subtle fade-in or slide-up using Intersection Observer or CSS `@keyframes` with `animation-play-state`. Keep them subtle; this is not a motion showcase.
- **Default export**, no required props
- Load Playfair Display and DM Sans from Google Fonts via a `<link>` tag in a wrapping component or via Tailwind font config
- Do not use any UI component libraries (no shadcn, no Radix, no Headless UI) — all components built inline with Tailwind
- No placeholder images — use text, typographic treatments, or color blocks for visual weight where images would otherwise go
